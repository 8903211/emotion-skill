# Data Preparation

Use this reference when the user needs to go from WeChat records to analyzable files.

## Authorized Sources Only

Accept only:

- exports from the user's own WeChat/Weixin account
- exports supplied by a participant with permission
- copied chat text or screenshots the user is allowed to share
- CSV/JSON/HTML/TXT generated by user-approved archive/export tools

Do not provide instructions for unauthorized account access, device compromise, password/key theft, or evading platform security.

## Preferred Export Shape

Best format:

```json
{
  "chat": "display name",
  "username": "stable id if available",
  "exported_at": "2026-01-03 09:32:25",
  "date_first_msg": "2026-01-01 00:14:54",
  "date_last_msg": "2026-01-03 09:01:21",
  "messages": [
    {
      "timestamp": "2026-01-03 08:12:00",
      "sender": "me",
      "type": "text",
      "content": "message text"
    }
  ]
}
```

Also support:

- JSON array of message objects
- JSONL, one message object per line
- CSV with columns like `timestamp`, `sender`, `content`, `type`
- TXT copied transcripts with timestamp/sender prefixes
- HTML exports when text can be extracted reliably

## Selection Workflow

1. List candidate chats with name, type, message count, first time, last time, and estimated attachment size.
2. Let the user select a small set:
   - relationship target
   - comparison chats
   - a date range or event window
3. Prefer exporting text-first data before heavy media.
4. For relationship analysis, start with:
   - conflict day plus 24 hours before and after
   - latest 3-7 days
   - one stable baseline window for comparison
5. For phrase coaching, export only the specific exchange plus enough prior context to understand tone.

## Hygiene Before Analysis

- Remove or mask phone numbers, addresses, account numbers, payment IDs, tokens, and precise locations unless needed for the analysis.
- Preserve timestamps and sender labels.
- Keep voice/media placeholders as placeholders unless transcripts are available.
- Mark missing media or voice content as evidence gaps.
- Exclude forwarded `[聊天记录]` blocks by default, because they can mix another conversation into the current relationship.

## Normalization

Use the script:

```bash
python scripts/normalize_chat_export.py input.json --output normalized.jsonl
```

For folders:

```bash
python scripts/normalize_chat_export.py exported_chats --output normalized.jsonl
```

The output is JSONL with:

- `timestamp`
- `sender`
- `type`
- `content`
- `source_file`
- optional `chat`, `raw_index`, `raw`

If normalization loses important fields, keep the original file path in the report and read the original as needed.
